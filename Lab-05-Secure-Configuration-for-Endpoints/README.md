# Lab 05 – Secure Configuration for Endpoints

## Overview

This lab demonstrates the implementation of secure endpoint configurations on an Ubuntu Linux system by applying security best practices aligned with industry-recognized security benchmarks such as the CIS Benchmarks. The lab focuses on auditing the system, disabling unnecessary services, enforcing strong password policies, and documenting configuration changes to improve the overall security posture.

---

## Objectives

- Apply secure configuration recommendations to an endpoint.
- Perform a security audit using Lynis.
- Identify and disable unnecessary services.
- Configure password complexity policies.
- Document security configuration changes.
- Improve endpoint hardening using security best practices.

---

## Lab Environment

| Component | Details |
|----------|---------|
| Operating System | Ubuntu 24.04 LTS |
| Platform | AWS EC2 |
| Security Audit Tool | Lynis |
| Configuration Files | /etc/ssh/sshd_config, /etc/security/pwquality.conf |
| Service Manager | systemd |

---

## Task 1 – Apply Security Benchmark

A security assessment was performed using the Lynis auditing tool to identify system hardening recommendations.

The SSH configuration was reviewed and updated according to secure configuration guidelines.

Recommended changes included:

- Disable direct root login.
- Verify SSH security settings.
- Review authentication configuration.
- Apply security recommendations generated by Lynis.

---

## Task 2 – Disable Unnecessary Services

Running services were reviewed using systemctl.

Non-essential services should be disabled to reduce the attack surface.

Example:

- Bluetooth service (if present)
- Printing services (if unnecessary)
- Other unused background services

---

## Task 3 – Configure Password Complexity

Password quality policies were configured using PAM.

The following password policy settings were applied:

| Parameter | Value |
|----------|-------|
| Minimum Length | 12 |
| Uppercase Characters | Required |
| Lowercase Characters | Required |
| Numeric Characters | Required |
| Special Characters | Required |

Example configuration:

```
minlen=12
dcredit=-1
ucredit=-1
lcredit=-1
ocredit=-1
```
